# AI Context - Sihame Content Engine

This file provides specialized context for AI coding assistants (like Gemini, Claude, or Copilot) to understand the project's unique architectural patterns, naming conventions, and logic flow.

## 🤖 Context for AI Agents

### 1. The Core Feedback Loop
- **REJECTED Status**: When a user rejects a draft, the backend updates the status in Supabase.
- **Feedback String**: Rejection reasons are stored in the `rejection_reason` column.
- **Regeneration**: In the frontend (`page.tsx`), the `handleRejectAndRegenerate` function calls the reject API and **immediately** triggers a new generation attempt using the initial prompt.

### 2. Naming Conventions & State
- `appState`: Tracks the current view in the UI:
  - `compose`: Initial input form.
  - `review`: Reviewing a generated `DRAFT`.
  - `approved`: Reviewing an `APPROVED` draft with adaptation options.
- `draftId`: The UUID of the current content draft in Supabase.
- `rawInput`: The initial title/topic provided by the user.

### 3. Repository Structure
- `ai_writer/backend`: FastAPI application.
- `ai_writer/frontend`: Next.js application.
- `ai_writer/knowledge_pack`: Brand identity documentation used for RAG-like context assembly.
- `ai_writer/DEPLOYMENT.md`: Details about Cloud Run and Vercel environments.

### 4. Special Logic (Platform Adaptation)
- After approval, a draft can be 'adapted' for different platforms (Instagram, TikTok, WhatsApp, Facebook).
- Each platform has its own system prompt segment located in the `workflow_service.py` or the `knowledge_pack`.

### 5. Multi-Environment (Dev/Prod)
- **Production**: Branch `main`, deployed to `c--sihame-content-engine`.
- **Development**: Branch `dev`, deployed to `sihame-backend-dev`.
- **CI/CD**: Managed via GitHub Actions (`.github/workflows/deploy-backend.yml`).
- **Secrets**: API keys (KIE_API_KEY, SUPABASE_*) are managed in GitHub Secrets and passed to Cloud Run.

### 6. Key Files for AI Agents
- `backend/app/services/workflow_service.py`: The heart of the business logic.
- `frontend/src/app/page.tsx`: The main workspace UI and state orchestrator.
- `backend/app/api/routes/generate.py`: API endpoints for content operations.
- `ai_writer/ARCHITECTURE.md`: High-level system interaction diagram.

> [!TIP]
> When modifying the backend, ensure all draft state transitions (`DRAFT` -> `REJECTED` -> `APPROVED`) are reflected both in the `WorkflowService` and the Supabase database.
