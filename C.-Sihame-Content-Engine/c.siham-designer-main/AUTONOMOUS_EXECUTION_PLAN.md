# AUTONOMOUS EXECUTION PLAN

## Purpose
Create a self-driven next-step ladder so the repo can keep improving without requiring manual prompting for every single move.

## Rule
Autonomous work here means:
- internal repo cleanup
- benchmark setup
- evaluation scaffolding
- documentation improvement
- small, reversible structural improvements

It does **not** mean:
- publishing client work automatically
- changing brand direction without review
- deleting major content without review
- large speculative rewrites without checkpoints
