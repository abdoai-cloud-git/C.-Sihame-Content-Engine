# Failure Pattern Report

## Purpose
Track recurring benchmark failures after generation and scoring.

## Candidate recurring patterns
- mood-first / meaning-second output
- generic editorial drift
- weak guided-practice embodiment
- educational under-clarity
- concept-contrast under-separation
- promo becoming too sales-like
- Arabic hierarchy too soft or too compressed
- decorative symbolism with weak semantic function

## Usage
After each benchmark cycle:
1. mark which patterns appeared
2. count frequency
3. prioritize only the top repeated issues
