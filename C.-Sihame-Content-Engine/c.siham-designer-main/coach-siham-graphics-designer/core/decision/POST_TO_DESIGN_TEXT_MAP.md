# POST TO DESIGN TEXT MAP

## Purpose
Connect post meaning to titling + on-image text selection.

This prevents random text placement.
It also decides the **font roles** early, so typography is part of the visual decision itself.

---

## Mapping logic

### Reflection post
- usually no aggressive headline
- choose 1 reflective line if needed
- keep design mostly paragraph-led or text-light

### Guided practice
- choose 1 short title
- choose 1-2 body lines with strong somatic meaning
- keep the rest in caption

### Educational post
- choose concept title
- choose 2-4 supporting points if legible

### Contrast post
- choose contrast headline
- each side gets short label only

### Promo post
- choose offer title
- supporting line = time / invitation / CTA

---

## Font decision rule at concept stage
Font choice must be decided during the design-decision stage, not added later as a vague styling hint.

For every post, decide explicitly:
- title font
- support/body font
- signature font
- whether any accent font is allowed at all

### Default logic
- title font: `Rubik`
- support/body font: `Rubik`
- signature font: `Rubik`

### Accent logic
- `Rakkas` is allowed only for the **title**
- only when the title is **3 words or less**
- and only when it improves the concept while staying premium and readable
- if there is any doubt, stay with `Rubik`

### Prompting rule
The image prompt should state the font roles clearly, for example:
- title font = Rubik
- support/body font = Rubik
- signature font = Rubik

or, when allowed:
- title font = Rakkas
- support/body font = Rubik
- signature font = Rubik

---

## Example for guided practice
Post meaning:
- inner speed
- slowing down
- self-contact
- safety message

Design text extraction:
- title: "رسالة أمان إلى الجسد"
- support line 1: "إذا شعرتِ اليوم أنكِ متسارعة من الداخل"
- support line 2: "المطلوب أن يصل جسدك رسالة أمان"

Why:
These lines preserve the emotional and practical center of the post without overloading the image.
